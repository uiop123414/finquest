## User Stories

### US-01 — Регистрация и вход

**Как** гость, **я хочу** зарегистрироваться по email и паролю, **чтобы** получить персональный доступ к системе учёта финансов.

**Критерии приёмки:**
- При регистрации с уникальным email создаётся аккаунт, возвращаются access + refresh токены
- При попытке зарегистрироваться с существующим email возвращается ошибка 409
- Пароль хранится в виде bcrypt-хэша (cost 12), никогда не возвращается в ответах
- После входа выдаётся JWT access token (30 мин) и refresh token (7 дней)
- При истечении access token клиент автоматически обновляет его через refresh

**Сценарий (Happy Path):**
1. Гость открывает `/login`, нажимает «Создать аккаунт»
2. Вводит email и пароль, отправляет форму
3. Система создаёт пользователя, возвращает токены
4. Клиент сохраняет токены в localStorage, перенаправляет на дашборд
5. В шапке отображается XP-бар: Lv.1, 0 XP

### US-02 — Добавление транзакции и получение XP

**Как** пользователь, **я хочу** добавить доход или расход с категорией, **чтобы** система учла его в аналитике и начислила мне опыт.

**Критерии приёмки:**
- Транзакция сохраняется с полями: сумма, тип (income/expense), категория, дата, заметка
- За каждую транзакцию начисляется +10 XP
- При добавлении первой транзакции автоматически выдаётся ачивка «Первый шаг»
- Транзакция немедленно отображается в таблице (без перезагрузки страницы)
- Дашборд пересчитывается при переходе на него

**Сценарий (Happy Path):**
1. Пользователь переходит в «Транзакции», нажимает «+ Добавить»
2. Заполняет форму: 5 000 ₽, «Расход», категория «Еда», дата сегодня
3. Нажимает «Сохранить»
4. Транзакция появляется в таблице, XP-бар обновляется (+10 XP)
5. На дашборде карточка «Расходы» увеличивается на 5 000 ₽

**Альтернативный сценарий (некорректные данные):**
1. Пользователь вводит сумму 0 или отрицательное число
2. Система возвращает 400 Bad Request
3. Форма отображает сообщение об ошибке, транзакция не создаётся

### US-03 — Постановка и выполнение финансовой цели

**Как** пользователь, **я хочу** создать финансовую цель с дедлайном и отслеживать прогресс, **чтобы** мотивировать себя откладывать деньги.

**Критерии приёмки:**
- Цель создаётся с: названием, целевой суммой, начальным накоплением, дедлайном
- Прогресс отображается в виде цветного прогресс-бара (синий → зелёный при 100%)
- Возможно пополнение накоплений по цели (сумма прибавляется к `current_amount`)
- При дедлайне < 30 дней — оранжевый индикатор; просрочено — красный
- Завершённые цели перемещаются в раздел «Выполнено»

**Сценарий (Happy Path):**
1. Пользователь нажимает «+ Добавить цель»
2. Вводит «Отпуск в Турции», 150 000 ₽, дедлайн через 3 месяца
3. Цель появляется с прогресс-баром 0%
4. Пользователь нажимает «+ Пополнить», вводит 30 000 ₽
5. Прогресс-бар обновляется до 20%
6. Когда `current_amount >= target_amount`, нажимает «Закрыть цель»
7. Цель перемещается в «Выполнено» с датой закрытия

### US-04 — Получение AI-совета по финансам

**Как** пользователь, **я хочу** получить персональный совет от AI на основе моих транзакций, депозитов и кредитов, **чтобы** улучшить управление деньгами.

**Критерии приёмки:**
- Совет генерируется на основе: транзакций за 30 дней, всех депозитов, кредитов, активных целей
- При наличии GEMINI_API_KEY или ANTHROPIC_API_KEY — ответ от реального LLM
- При отсутствии ключей — rule-based ответ (работает всегда)
- Совет содержит конкретные цифры из данных пользователя
- Время ответа ≤ 3 сек для rule-based, ≤ 10 сек для AI

**Сценарий:**
1. Пользователь на дашборде нажимает «Получить совет»
2. Кнопка меняется на «Анализирую...»
3. Система собирает финансовый контекст из БД
4. Запрашивает LLM или использует rule-based логику
5. Отображает совет из 4–5 конкретных пунктов

### US-05 — Импорт транзакций из CSV

**Как** пользователь, **я хочу** загрузить CSV-файл с историей операций, **чтобы** быстро наполнить систему данными без ручного ввода.

**Критерии приёмки:**
- Формат файла: `date,amount,type,note` (заголовок обязателен)
- Категория определяется автоматически по ключевым словам в `note`
- Дубликаты игнорируются (проверка по `external_id`)
- За каждую импортированную транзакцию начисляется +5 XP
- Результат: «Импортировано X / Y»

### US-06 — Просмотр и управление депозитами

**Как** пользователь, **я хочу** добавить банковский депозит и видеть прогнозируемый доход, **чтобы** оценивать доходность своих вложений.

**Критерии приёмки:**
- Депозит хранит: банк, сумму, ставку, даты открытия/закрытия, заметку
- Годовой доход рассчитывается автоматически: `amount × rate / 100`
- Карточка показывает количество дней до закрытия (оранжевый < 30, «истёк» если прошло)
- Сводные карточки: итого в депозитах, суммарный годовой доход
